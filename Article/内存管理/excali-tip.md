# 内存分配 配图提示词

遵照 excalidraw-diagram skill。以下用于 `Article/内存管理`，与 `内存分配.md` / `内存分配源码1.md` / `内存分配源码2.md` 配套；偏仓库比喻，少堆术语。

---

## 结构

### 1. 仓库角色关系总览（主图）

- **图类型**：层级图 + 关系图（Relationship / 层级）。
- **画面元素**：
  - 顶层：`mheap`（总仓），下面挂一排 `mcentral`（补给站，可画成「按尺码分栏」）。
  - 左侧：`mcache`（口袋）绑在 `P` 上，箭头指向 mcentral 「无空间则去领 mspan」。
  - 中间：`mspan`（货架/托盘），框内写「一格一格小槽 或 整段大对象」。
  - 旁注小框：`pageAlloc`（页账本）贴在 mheap 旁；`heapArena`（户口簿）画成「街区格子 + 小册子」图标。
  - 角落：`fixalloc`（行政库房）单独小框，虚线连到「mspan/mcache 元数据」，标注「分配内存」。
  - 角落：`size class`，简表 `class` / `bytes/obj`（如 `1→8`、`2→16`）即可。

- **作用（写在各框旁）**：
  - **`mheap`**：全堆总管；管 span、跟系统要内存/还内存。
  - **`mcentral`**：每一档尺码一个；这一档所有 `mspan` 都归它管，`mcache` 不够了来它这儿领。
  - **`mcache`**：每个 `P` 一个；记「这一档当前正用哪块 span」，小对象多半在这儿就分完、不用锁。
  - **`mspan`**：一段连续内存的「一块板」；小对象切成很多格，大对象可独占整块。
  - **`pageAlloc`**：按**页**记账：哪儿空、能连续拿多长，给拆 span、大块用。
  - **`heapArena`**：大块地址对应的目录，记每页属于谁、方便 GC 查。
  - **`fixalloc`**：给分配器**自己**用的定长小块内存，不走普通堆那套。
  - **`size class`**：小对象归到固定几档尺寸里，再去找对应档的 central / span。

## 流程

### 1. 分配总览

- **图类型**：组合流程图（上半截分叉 + 下半截左右两路或泳道），一张画布内完成。

- **建议布局（自上而下）**：
  1. **最上**：圆角框 `mallocgc`（堆分配入口）。
  2. **第一层分叉**：菱形 `size == 0?` → 是：短箭头到 `zerobase`（靠边即可，不占主视觉）。
  3. **第二层分叉**：菱形 `大对象?`（或写「超过小对象上限」）→ **左右分栏**（或上下两带），避免和下面竖条打架。

- **左栏 / 上带 —— 小对象主路径（投胎之旅四步，竖着一条）**  
  自上而下串联四格，箭头向下：  
  `mcache`（口袋，无锁）→ `mcentral`（补给站，加锁批货）→ `mheap` + `pageAlloc`（总仓拆页）→ `OS` → `heapArena`（登记）。

- **小对象栏角落嵌小图 —— tiny**  
  在四步竖条**左侧或左上角**加一个小方框：「16 字节微盒」内画 3～4 个色块拼在一起；箭头指向 `mcache`，旁注：**极小 noscan 先拼单，再不够才占整格**。  
  另用一行小字即可：`noscan` / `scan` 在入口已分流，不必再画两条粗线（避免太碎）。

- **右栏 / 下带 —— 大对象短路**  
  粗箭头：`P` → `mheap`（按页，`pageAlloc`）→ `OS`（需要时）。  
  旁注：**不经 `mcentral`**、**不按小档槽位切**（可与「小对象四步」形成视觉对比）。

- **目的**：一张图同时回答三件事——有没有零长、大件和小件走法差在哪、小件典型四步 + tiny 例外长什么样。
