## 5.1 事务模型与 ACID

### 1. ACID 四块拼图（关系图 + 层级图）

- **图类型**：关系图（Relationship）或中心辐射思维导图。
- **中心**：`事务 Transaction`。
- **四象限/四分支**（每块一个色块卡片）：
  - **A 原子性**：旁挂小框 `Undo Log`，箭头标注「翻车则按日志回滚」。
  - **C 一致性**：旁挂 `约束/业务规则`，标注「目标是结果合法，靠应用 + 约束兜底」。
  - **I 隔离性**：双线指向 `锁` 与 `MVCC`（可虚线连到 5.2）。
  - **D 持久性**：旁挂 `Redo Log` + `WAL`，标注「先记日志再刷页」。
- **图注**：ACID 里 C 常被说成「目标」，A/I/D 是手段之一。

### 2. 隔离级别阶梯（对比图 Comparison）

- **图类型**：对比图 / 纵向阶梯。
- **布局**：自下而上或自上而下四级台阶：`RU` → `RC` → `RR` → `Serializable`。
- **每级旁注**（短标签）：
  - `RU`：脏读风险高。
  - `RC`：每次读可能看到新提交 → 不可重复读。
  - `RR`：InnoDB 默认；旁注「快照 + 锁配合」。
  - `Serializable`：读也加锁 → 最严、并发最差。
- **可选**：右侧一列「异常」小图标区（用文字框代替图标）：脏读、不可重复读、幻读，用虚线连到「在哪个级别仍可能出现」。

## 5.2 MVCC 多版本并发控制

### 1. 读写冲突：两条路（对比图）

- **图类型**：对比图（左右两栏）。
- **左栏「武斗」**：`写进行中` → `读等待`（锁/阻塞），标注「当前读、串行感」。
- **右栏「文斗」**：`写产生新版本` → `读看旧版本`（MVCC），标注「快照读、少阻塞」。
- **中间**：同一行数据画成「当前行 + 虚线老版本」示意即可。

### 2. 快照读 vs 当前读（流程图）

- **图类型**：小型流程图。
- **入口**：`SELECT`（菱形：是否 `FOR UPDATE` / `LOCK IN SHARE MODE` / DML）。
- **分支**：
  - **快照读**：走 `ReadView + 版本链`。
  - **当前读**：走「读最新 + 可能加锁」。
- **旁注**：列出 `INSERT/UPDATE/DELETE`、`* for update` 等到「当前读」一侧。

### 3. 版本链（关系图）

- **图类型**：关系图（链式）。
- **画面**：一行记录当前版本矩形，左侧或上方用箭头串联多个「旧版本」矩形（从新到旧）。
- **每版本框内小字**：`trx_id`、`roll_pointer`（指向前一版本）。
- **最旧端**：接 `Undo` 存储区小圆柱/文件夹图形。
- **图注**：`insert undo` 与 `update undo` 可用两种描边颜色区分，旁注「谁可能被别事务的快照依赖」。

### 4. ReadView「考勤快照」判断（流程图）

- **图类型**：流程图。
- **顶部**：快照框内列字段：`m_ids`、`min_trx_id`（或 up_limit）、`max_trx_id`（或 low_limit）——与正文表格一致即可。
- **中间**：菱形序列，对某条版本的 `trx_id`：
  - 小于 `min_trx_id` → 可见。
  - 大于等于 `max_trx_id` → 不可见。
  - 否则 → 若在 `m_ids` → 不可见；否则 → 可见。
- **出口**：`沿 roll_pointer 找下一版本` 循环（可用虚线回到判断菱形）。
- **禁止**：不要用 Emoji；可用「打卡/下班」文字比喻写在旁注框（浅黄）。

### 5. RC vs RR：何时拍 ReadView（时间线图）

- **图类型**：时间线图（Timeline）。
- **横轴**：时间，标 `T1` 第一次查询、`T2` 他事务提交、`T3` 第二次查询。
- **上下两轨对比**：
  - **RC**：在 `T1`、`T3` 各画一个相机「重拍 ReadView」，第二次可能看到新数据。
  - **RR**：只在 `T1`（或首条一致性读）拍一次，后面查询共用同一张快照。
- **图注**：对应正文「进门拍一张 vs 每次查重拍」。

### 6. RR 与幻读：情况 A / 情况 B（对比图）

- **图类型**：对比图。
- **共同前提**：两次范围查询之间，别事务插入新行。
- **情况 A**：仅 `快照读` → 第二次仍「看不到新行」，旁注「MVCC 快照不变」。
- **情况 B**：第二次为 `当前读`（或先更新再查）→ 需要 `间隙锁/临键锁` 等，旁注「当前读走锁，和纯快照路径不同」。
- **底栏**：一句话收束「防幻读是 MVCC + 锁分工，不是单靠 MVCC 包打天下」。
