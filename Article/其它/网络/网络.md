## 1. 分层与总览

OSI/TCP-IP 对照、各层典型协议一张表、封装与跳数概念

**规划：共 1 篇**

1. **《OSI 与 TCP/IP：分层对照 + 各层典型协议一览》**
   - **模型与表**：OSI 七层与 TCP/IP 四层（或五层教材写法）的对应关系；每层列 2～4 个最常见协议做「地图」（如 HTTP/DNS、TCP/UDP、IP/ICMP、ARP、以太网），方便以后查书、对抓包树状结构心里有数。
   - **用「层」拆问题（直觉）**：示例式说明先判哪一层——应用协议与 DNS；连接与端口多半在传输；跨网段与路由在网路；同一二层域内 MAC/交换机问题在链路；只建立分层排查的框架，不展开各协议细节。
   - **封装与 PDU（略讲）**：数据向下走时逐层加首部、各层 PDU 习惯称呼（如 TCP 段、IP 报文、以太帧）；点到为止，不讲各首部每个字段。
   - **逐跳与端到端（略讲）**：哪些东西通常只在一跳内有意义（如 MAC）、哪些通常端到端保持（如 TCP 连接、IP 地址不变下跨路由器）；与后面传输/网路/链路各篇轻轻挂钩即可。

## 3. 传输层

TCP 握手挥手与状态、可靠性/拥塞、UDP、选型

**规划：共 3 篇**

1. **《TCP 连接：三次握手、四次挥手与状态机》**
   - **握手/挥手**：三次握手各自交换什么、为何需要；四次挥手与半关闭直觉；与 SYN/ACK/FIN 在抓包里的对应。
   - **状态机提要**：LISTEN、ESTABLISHED、TIME_WAIT、CLOSE_WAIT 等常见状态在什么场景出现；TIME_WAIT 常被问到时可给一句原因级解释。
   - **边界**：不讲内核实现细节；误配置与排错可一笔带过指向运维场景。
2. **《TCP 可靠性与拥塞控制（概念篇）》**
   - **有序与确认**：序号与确认号、超时重传、重复与去重的直觉；累积确认概念。
   - **流量控制**：滑动窗口在「别让发送方撑爆接收方」里的作用（点到为止）。
   - **拥塞控制**：慢启动、拥塞避免等名字的动机级解释；**不写**具体算法推导与参数调优全书。
   - **可选**：抓包或时序图一句话彩蛋（附录级，非正文必需）。
3. **《UDP 与传输层选型》**
   - **UDP 长什么样**：无连接、尽力交付、头部单薄；与「谁保证可靠性」的分工（应用或下层协议）。
   - **典型用途**：DNS 查询、实时音视频、游戏等为何常选 UDP（各一句）。
   - **选型**：何时倾向 TCP、何时 UDP、QUIC 在什么意义上补缺；与第 2 篇应用层长连接可呼应一句。

## 4. 网络层等

IP/CIDR、路由与最长前缀、ARP、ICMP、NAT；

**规划：共 1 篇**

1. **《IP 与 CIDR：地址、子网与最长前缀匹配》**
   - **IPv4 地址与掩码**：网络位与主机位、子网划分粗糙直觉；广播域与路由的关系一句话。
   - **CIDR 表示法**：前缀长度、路由聚合（路由汇总）为何省表项。
   - **最长前缀匹配**：路由器如何从多张候选路由里选出一条；与「默认路由」的配合。
   - **边界**：具体厂商 CLI 配置点到为止；IPv6 可一节简略带过或注明另文。
2. **《ARP 与 ICMP》**
   - **ARP**：同一二层域内「已知 IP 求 MAC」的步骤；缓存表、超时；与跨网段时走网关的直觉衔接。
   - **ICMP**：差错报告与诊断（如目标不可达、TTL 超时）；ping（Echo）在测什么、不能推出什么。
   - **排错**：链路上常见现象与这两者的关系（不写防火墙规则大全）。
3. **《NAT：原理、类型与对上层的影响》**
   - **动机**：私网地址与地址短缺；改写什么（通常 IP/端口）。
   - **SNAT/DNAT**：出站与入站视角；端口映射、静态映射直觉。
   - **对上层**：为何有时 FTP/P2P/游戏要特殊处理；与 TCP 连接表、会话状态的粗关系。
