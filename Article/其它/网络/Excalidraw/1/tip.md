对应正文：`1-分层对照与协议一览.md`。建议画布约 **1200×800**，手写字体、中文标签；图中**不要用 Emoji**。

## 图 1：OSI 七层 ↔ 五层 ↔ TCP/IP 四层（对照）

| 项目 | 说明 |
|------|------|
| **建议类型** | 对比图（Comparison）或 层级图（Hierarchy） |
| **建议文件名** | `1-OSI-五层-TCP对照.md` |
| **插入位置** | §2「OSI、五层与 TCP/IP 的对应关系」附近 |

**提示词（给 AI 画图 / 自己照着画）：**

> 画一张**三列对照表**，左列「OSI 七层」从下到上：物理、数据链路、网络、传输、会话、表示、应用（或从上到下与教材一致，三列同一方向即可）。中列「五层」：物理、数据链路、网络、传输、应用；用**大括号或色块**把 OSI 的会话+表示+应用 合并标成「应用层」。右列「TCP/IP 四层」：网络接口层「含物理+链路」、网际层、传输层、应用层。用**箭头或同色背景**标出三列之间的对应关系。风格：简洁矩形卡片、浅色区分列，标题字号最大。

## 图 2：五层协议栈 + 各层代表协议（竖向「地图」）

| 项目 | 说明 |
|------|------|
| **建议类型** | 层级图（Hierarchy） |
| **建议文件名** | `1-五层协议栈与代表协议.md` |
| **插入位置** | §3「各层协议地图」 |

**提示词：**

> 画一个**自下而上**的五层栈：最下「物理层」，其上「数据链路层」，再「网络层」，再「传输层」，最上「应用层」。每一层右侧或层内用小标签列出**代表协议**：应用层写 HTTP、DNS；传输层 TCP、UDP；网络层 IPv4、ICMP；链路层 以太网、ARP；物理层写 网线/光纤/Wi‑Fi 任选其一作代表。层与层之间用横线分隔。可加一句旁注：「Wireshark 展开顺序常与自下而上对应：Frame「链路」→ IP → TCP/UDP → 应用」。配色：每层不同浅色底。

## 图 3：封装（套娃）与各层 PDU 名称

| 项目 | 说明 |
|------|------|
| **建议类型** | 层级图 / 洋葱剖面（Freeform） |
| **建议文件名** | `1-封装与PDU套娃.md` |
| **插入位置** | §5「封装与各层 PDU 叫法」 |

**提示词：**

> 画**横向或纵向「套娃」**：最内层小矩形写「应用数据」，外包一层标注「TCP 段 / UDP 报文」，再外包「IP 数据报」，再外包「以太网帧」，最外可画虚线表示「比特流」。每层旁标注 PDU 习惯称呼：**段/数据报、分组、帧、比特**。箭头标注方向「发送端：向下封装；接收端：向上解封装」。风格：嵌套矩形，由深到浅或由内到外颜色渐变。

## 图 4：逐跳 vs 端到端（MAC / IP / TCP）

| 项目 | 说明 |
|------|------|
| **建议类型** | 关系图（Relationship） |
| **建议文件名** | `1-逐跳与端到端-MAC-IP-TCP.md` |
| **插入位置** | §6「逐跳与端到端」 |

**提示词：**

> 画**从左到右**拓扑：主机 A → 路由器 R1 → 路由器 R2 → 主机 B，设备用圆角矩形，链路用线段。  
> **对每一跳**（A–R1、R1–R2、R2–B）在链路旁标注「源 MAC / 目的 MAC **会变化**」或画小图标表示「每跳重写」。  
> **在 IP 层**用一条贯穿的虚线框或同色条带标注「源 IP、目的 IP **通常不变**「无 NAT 时」」。  
> **在 TCP 层**在 A 与 B 上方画一条「逻辑连接」双箭头，标注「端口号、TCP 连接 **端到端**；中间路由器不参与握手」。  
> 图例：三种颜色分别对应二层/三层/四层说明。文字简短，避免挤成一团。
