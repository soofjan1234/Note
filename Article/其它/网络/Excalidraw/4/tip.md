192.168.1.0   |   255.255.255.0   |   On-link   |   WLAN   |   在自己家局域网内直接找

下面是最长前缀匹配

假设路由表里有这两条记录：

路由 A：192.168.1.0/24 （前缀长度 24，掩码 255.255.255.0）
路由 B：192.168.1.128/25 （前缀长度 25，掩码 255.255.255.128）
现在来了一个数据包，目标 IP 是：192.168.1.130。

匹配路由 A 吗？ 匹配。192.168.1.130 属于 192.168.1.0 这个大网段。
匹配路由 B 吗？ 也匹配。192.168.1.130 刚好就在 192.168.1.128 开始的那个更小的网段里。
结论：路由器会选择 路由 B。因为 /25 比 /24 匹配的位数更多（前缀更长），这意味着路由 B 给出的路径更精确。

## 2. ARP (地址解析)
- **绘图逻辑**：**广播问询 + 单播回复**。
- **元素建议**：
    - 主机 A 拿扩音器大喊：“谁是 `1.1.1.1`？”（波纹扩散状，表示广播）。
    - 交换机里的流量横切局域网所有设备。
    - 主机 B 单独拉一条细箭头回传给 A：“是我，我的 MAC 是 `BB`。”（定向线条）。
- **欺骗逻辑**：黑客在旁边画个红叉，强行插话：“我是网关，发给我！”

## 3. NAT (地址转换)
- **绘图逻辑**：**内外隔离 + 端口映射表**。
- **元素建议**：
    - 画一道墙（代表网关/路由器），左边是私网（多台设备，IP 类似 `192.168.x.x`），右边是公网（一个 IP）。
    - 关键在于**数据包变色/变换**：进入墙前源地址是私网 IP，穿过墙后源地址变成了公网 IP。
    - 墙中间悬浮一张“记录表”，画出端口对应关系。

## 4. ICMP (控制消息)
- **Ping**：画两个球，一个丢出去（Request），一个弹回来（Reply）。
- **Traceroute**：画一排门（路由器），第一个包 TTL=1 到一号门碎掉并回传信息；第二个包 TTL=2 穿过一号门到二号门碎掉... 就像**探测地雷**一样分步推进。


### 2.1 IP（补充）
- **绘图逻辑**：IP 负责“跨网寻址”，子网掩码负责“网络号/主机号分界”。
- **元素建议**：
    - 画一个 IP 地址拆分条：`192.168.1.10/24`，前 24 位标蓝色（网络号），后 8 位标绿色（主机号）。
    - 旁边放一条对照：`/24 = 255.255.255.0`，强调 CIDR 与掩码等价。
    - 再画两个地址示例：`192.168.1.10` 与 `192.168.1.88`（同网段，可直连）；`192.168.2.10`（跨网段，需走网关）。
    - 用一句结论标注：**同网段看 MAC，跨网段先找网关再转发**。

### 4.2 NAT 穿透（补充）
- **绘图逻辑**：两端都在 NAT 后，通过 STUN 打洞，失败走 TURN 中继。
- **元素建议**：
    - 左右各画一个 NAT（A 侧、B 侧），中间上方画 STUN，下方画 TURN。
    - 第一步：A/B 各自向 STUN 注册，拿到各自公网 `IP:Port`。
    - 第二步：双方交换对端公网地址后“几乎同时发包”打洞。
    - 成功路径：A <------> B（直连粗箭头）。
    - 失败路径：A -> TURN -> B（虚线或橙色箭头，标注“中继，延迟更高”）。

### 4.3 NAT 类型（补充）
- **绘图逻辑**：按“放行范围”从宽到严排序，突出对穿透成功率影响。
- **元素建议**：
    - 四个卡片并排：Full Cone / Restricted Cone / Port Restricted / Symmetric。
    - 每个卡片画“内网主机 -> 公网映射”与“谁能回包”的规则。
    - 在最右侧 Symmetric 卡片加红色警示：**打洞最难，常需 TURN**。
    - 底部加一条总结：`越严格 -> P2P 成功率越低 -> 中继概率越高`。
