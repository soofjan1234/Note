## 图 5：输入网址到显示网页（§5 终极串联题）

对应正文：`1-分层对照与协议一览.md` §5「输入网址到显示网页的完整过程」（约 L78–L107）。建议画布 **1200×800**，可画成**横向时间线**或**双泳道（客户端 | 服务端）**；图中不要用 Emoji。

| 项目 | 说明 |
|------|------|
| **建议类型** | 时间线图（Timeline）或 流程图（Flowchart）+ 局部「套娃」放大框 |
| **建议文件名** | `1-从URL到页面-全流程.md` |
| **插入位置** | §5 标题下第一段之后 |

**总提示词（整图骨架）：**

> 共 6 个阶段：**DNS → TCP「+TLS」→ 封装发送 → 路由转发 → 服务端解包与响应 → 浏览器渲染与挥手**。每个阶段的标题用**色块卡片**包住并放左上角，然后阶段之间用黑色虚线分割。

---

### 5.1 URL 解析与 DNS（步骤 1）

- **绘图逻辑**：「浏览器 → 多级缓存 → 递归/迭代 DNS → 得到 IP」一条链。
- **元素建议**：
  - 左起画浏览器矩形，箭头依次经过小框：`浏览器/进程缓存（若有）、OS DNS 缓存、hosts` → `递归解析器` → `根服务器` → `TLD 服务器`  → `权威 DNS` → 终点写 `IP`。
  - URL 条单独一行：`https` 与 `www.example.com` 用不同颜色高亮协议与主机名。

### 5.2 TCP 与 TLS（步骤 2）

- **绘图逻辑**：先画 **SYN → SYN+ACK → ACK** 三个小包序号以及箭头；下方画 **TLS 握手**小框，也是箭头，交换证书，算出密钥，加密。
- **元素建议**：
  - 客户端随机端口 `ephemeral` 与服务器 `443` 标在 TCP 头示意旁。
  - TCP 建连完成后，用**括号或下挂子流程**接 TLS，表示「在已建立的 TCP 上叠加」。

### 5.3 封装 HTTP 并发出（步骤 3）

- **绘图逻辑**：**纵向套娃**；与图 3 呼应但这里强调「发往网关」。
- **元素建议**：
  - 内层写 `HTTP Request`，外包 `TCP Segment`「端口+序号」，再包 `IP Datagram`「源目 IP」，最外 `Ethernet Frame`「目的 MAC = 网关」。
  - 单独画小插图：`ARP` 问号广播 → 网关 MAC 回答 → 帧头目的 MAC 填好。

> 套娃做到 以太网帧 这一层时，帧头里要写 目的 MAC。
> 但你的电脑只知道 目标 IP（比如 example.com 的服务器 IP），不知道下一跳网卡的 MAC。
> 
> 所以多了一步：
> 
> 先问网关：「默认网关是谁？它的 MAC 是多少？」
> 这就是 ARP：在同一局域网里广播问「谁是这个 IP 的 MAC」，网关回答。
> 然后你才能把帧的 目的 MAC 填成网关，帧才能发到路由器

### 5.4 路由转发（步骤 4）

- **绘图逻辑**：**逐跳**展示「拆到网络层看目的 IP → 查表 → 改二层 MAC → 下一跳」。
- **元素建议**：
  - 画 3 台路由器串联，每台上方悬浮小表「路由表」；包经过时每跳旁注「目的 MAC 变」「目的 IP 不变」。
  - 其中一跳可标注 `LPM` 或「最长前缀匹配」缩写，避免长句。

> 数据离开你家路由器之后，会在很多台路由器之间跳。每一跳大致是：
> 
> 收到帧（二层）
> 拆开看 IP（三层）：看「目标 IP 是谁」
> 查路由表：决定「下一跳从哪个口出去、下一台路由器是谁」
> 重新封二层：目的 MAC 改成下一跳的 MAC（源 MAC 也改成本路由器该口的 MAC）
> 再发出去

### 5.5 服务端解包与回包（步骤 5）

- **绘图逻辑**：与发送端**镜像**：物理 → 帧 → IP → TCP → HTTP，箭头方向反转为「入站解封装」；再画一条对称箭头 `HTTP Response` 回程。
- **元素建议**：
  - 服务器侧画 `Nginx/Tomcat` 小进程框，HTTP 箭头指向它。
  - 回程可与去程用**上下双轨**或不同颜色区分，避免箭头交叉难读。

### 5.6 浏览器渲染（步骤 6）

- **绘图逻辑**：`HTML` → `DOM/CSSOM` → `子资源请求「JS/CSS/图」` 简化为三个小框。
- **元素建议**：
  - 子资源可用虚线小箭头从主页面框引出，表示「并发再请求」。
  - Keep-Alive 用一句旁注小字：`连接复用` 或 `可多条请求共连接`，不写长解释。
