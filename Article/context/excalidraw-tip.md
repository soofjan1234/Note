## 2. Value：代码示例 + 向上查找（近到远、找不到为空）
请绘制「Value 代码示例与查找」示意图，严格覆盖 `Article/context/context.表文件.md` 中 Value 相关内容：
1. 展示 Value 代码示例表达的流程：在函数里把用户信息保存到 Context 中；并在 goroutine 中通过 Value(key) 读取
2. 说明 Value 向上查找：从当前这一层开始；当前层有设置就直接用；没有就去父上下文找，直到最外层
3. 如果一路都没找到，就会得到空结果
4. 强调一句：离你最近的那次设置优先生效
5. 额外补充：Value 还有个功能是查找“最近可取消祖先”；当系统需要把“取消”传给子上下文时，会向上找一个负责响应取消的父级，并选择离当前最近的那一层；如果中间某层把取消传递关掉了，向上查找可能被截断，子上下文可能就收不到父取消；提示：普通业务只需关心“取消会沿父链传下去”，不需要自己去用 Value 做“最近可取消祖先”的查询

画面布局建议：
1. 上半部分左侧画“代码示例流程”：输入 -> 保存到 Context -> goroutine 里读取 Value
2. 上半部分右侧画“查找路径”：Current（当前层）-> Parent（父）-> Root（最外层）
3. 用高亮从 Current 指向“最先命中的那一层”，旁边写：优先生效
4. 在链末端加一个“找不到 = 空结果”的小注释框

## 3. Cancel：概念 + 父取消子一起停 + 接入规则与用法
请绘制「Cancel 心智模型」图，严格覆盖 `Article/context/context.表文件.md` 中 Cancel 相关内容：
1. 取消就是“让一件事停下来”
2. 取消来自手动取消或时间到了（截止/超时）
3. 当父上下文取消后，子上下文会一起进入取消状态，帮助下游停止工作并做收尾
4. 创建可取消的子上下文：父取消 -> 子跟着取消；也支持自己手动取消子级立刻停下
5. 向上查找最近可取消祖先：从近到远找离你最近的、确实能响应取消的父级，然后把自己接到它那里
6. 若中间层断开取消传递，向上查找可能被截断，子可能收不到父取消
7. 两条兜底路径：父支持回调型取消接入（跑回调并在回调里把子标记为取消）；无法挂到父的 children 上就只能起一个 goroutine 等取消
8. 用法：把取消当刹车；先等再收尾；如果没停住先检查是不是中间层断开了取消传递

画面布局建议：
1. 顶部画“取消来源”模块：手动取消 与 时间到了（deadline/timeout）
2. 中间画一条父 -> 子 的关系链，标注：父取消 -> 子进入取消状态 -> 下游停止 + 收尾
3. 下半部分画“接入规则”流程：向上查找最近可取消父级；同时在旁边画一个“可能截断/可能收不到取消”的分支点
4. 最底部画“三句用法”小卡片：刹车 / 先等再收尾 / 没停住检查断开取消传递
